# Phase 7: Documentation & Release Report

**Status:** Ready for Execution
**Goal:** Package the validated software into trusted, signed artifacts and provide comprehensive documentation for end-users.

## 7.1 Release Artifacts & Signing

### Software Bill of Materials (SBOM)
*   **Goal:** Supply Chain Security.
*   **Action:** Generate a `cyclonedx` or `spdx` file listing every crate and npm package used.
*   **Command:** `cargo sbom` / `syft`.

### Code Signing
*   **Windows:**
    *   **Cert:** EV (Extended Validation) Code Signing Certificate.
    *   **Verify:** Right-click .exe -> Properties -> Digital Signatures.
*   **macOS:**
    *   **Cert:** "Developer ID Application".
    *   **Notarization:** Use `xcrun notarytool`. Must pass Apple's server check.
*   **Android:**
    *   **Key:** Release Keystore (`.jks`).
    *   **Check:** `apksigner verify --verbose my-app.apk`.

### Release Manifest
*   **Checksums:**
    *   Generate `SHA256SUMS.txt`.
    *   `sha256sum Noteece_Setup_x64.exe >> SHA256SUMS.txt`
*   **Changelog:**
    *   Extract section from `CHANGELOG.md`.
    *   Format as GitHub Release notes.
